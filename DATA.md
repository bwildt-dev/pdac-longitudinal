# Data

The framework does not ship any patient data. You provide the CT volumes and a
labels CSV; everything else (segmentations, ROI masks, features, cache) is
generated by `pdac_longitudinal preprocess`. This document describes the
formats it expects and produces.

## Input NIfTI tree

`preprocess` reads NIfTI volumes from `data.nifti_dir`, laid out by phase,
patient, and timepoint:

```
<nifti_dir>/
  <phase>/                 # arterial, venous, ...
    <patient_id>/
      t0/  *.nii.gz        # baseline scan(s)
      t1/  *.nii.gz        # post-neoadjuvant scan(s)
```

- `phase` is one of the CT contrast phases in `data.phase` (a preference order:
  each patient uses the first phase that has both a t0 and a post-NAT scan).
- `t0` is the baseline timepoint; the post-NAT timepoint is any tag in
  `data.post_nat_tps` (default `[t1]`).
- Multiple `*.nii.gz` per timepoint directory are allowed; the discovery step
  picks one T0/T1 pair per patient.

This is the output layout of [dataset-composer](https://github.com/bwildt-dev/dataset-composer),
which assembles it from DICOM. You can produce it any other way as long as the
tree matches.

## labels.csv

A single CSV keyed by `patient_id`. Four columns are reserved; every other
column is treated as a clinical covariate.

| Column        | Type   | Meaning                                             |
|---------------|--------|-----------------------------------------------------|
| `patient_id`  | str    | Matches the `<patient_id>` directory name.          |
| `cohort`      | str    | Cohort label, used for filtering and stratification.|
| `time_months` | float  | Time to death or last follow-up, in months.         |
| `status`      | 0/1    | Event indicator: 1 = death observed, 0 = censored.  |

`(time_months, status)` are the survival target. All remaining columns are read
as clinical features (numeric). Missing values are median-imputed per fold;
optionally, a `<col>__isna` indicator is added when
`data.clinical_missingness_flags` is on.

## CT cache (.npz)

`preprocess` writes one compressed `.npz` per patient under `data.cache_dir`.
CT arrays are float32, masks are uint8, all resampled to
`preprocessing.target_spacing` and cropped/padded to `preprocessing.patch_size`
(default 1.5 mm isotropic, 160^3).

| Key                          | Dtype | Description                                  |
|------------------------------|-------|----------------------------------------------|
| `t0`, `t1`                   | f32   | Normalised CT patches, baseline and post-NAT.|
| `mask_it`, `mask_it_t1`      | u8    | Intra-tumoural mask, T0 and T1.              |
| `mask_pt1/2/3`, `_t1`        | u8    | Peritumoural rings (inner to outer).         |
| `mask_tvi`, `mask_tvi_t1`    | u8    | Tumour-vessel interface mask.                |
| `liver_t0/t1`, `pancreas_t0/t1`, `kidneys_t0/t1` | u8 | Organ masks used for alignment and features. |
| `valid_t0`, `valid_t1`       | u8    | 1 = real content, 0 = padding.               |
| `phase_used`                 | bytes | Resolved CT phase for this patient.          |
| `anatomy_features_json`      | bytes | JSON-encoded anatomy feature dict.           |
| `vessel_features_json`       | bytes | JSON-encoded vessel-tumour feature dict.     |
| `radiomic_features_json`     | bytes | JSON-encoded radiomics (added by the radiomics step, if enabled). |

## Feature vectors (model inputs)

At training time each cached patient becomes a sample with these tensors, whose
dimensions are fixed by the schema and gated by the `modules` toggles:

| Tensor     | Dim   | Source                                              |
|------------|-------|-----------------------------------------------------|
| `clinical` | var   | Non-reserved columns of `labels.csv`, per fold.     |
| `anatomy`  | 31    | Per-organ volume/HU and deltas, from segmentations. |
| `vessel`   | 78    | Vessel-tumour contact and interface metrics.        |
| `radiomic` | 2247  | Per-compartment PyRadiomics features (T0, T1, delta).|

`duration`/`event` come from `time_months`/`status`. A module set to `false`
under `modules` contributes no token, and its feature dimension is treated as 0
at runtime.
